# C++智能指针的使用与注意事项

- 参考文献  [万字长文全面详解现代C++智能指针：原理、应用和陷阱 - 七昂的技术之旅 - 博客园](https://www.cnblogs.com/qiangz/p/17911186.html)

## 一.概述

### a.程序对象的生存期

- 全局对象在程序启动时分配，在程序结束时销毁。
- 对于局部自动对象，当我们进入其定义所在的程序块时被创建，在离开块时销毁。
- 局部static对象在第一次使用前分配，在程序结束时销毁。
- C++还支持动态分配对象。**动态分配的对象的生存期与它们在哪里创建是无关的，只有当显式地被释放时，这些对象才会销毁。**

### b.智能指针概述

- 智能指针是C++中用于自动管理动态分配内存的类模板，它们通过RAII（资源获取即初始化）技术确保在适当的时候自动释放内存，从而避免内存泄漏。





## 二.主要智能指针类型

### 1. `std::unique_ptr`

**特点**：独占所有权，不可复制，只能移动

**a.基本使用**

```cpp
#include <memory>
#include <iostream>

// 创建unique_ptr
std::unique_ptr<int> p1(new int(42));
auto p2 = std::make_unique<int>(100);  // 推荐使用make_unique

// 访问指针
std::cout << *p1 << std::endl;  // 解引用   输出结果：42
std::cout << *p2 << std::endl;  // 解引用   输出结果：100
std::cout << p1.get() << std::endl;  // 获取原始指针   输出结果：0000022A78D67E70

// 转移所有权
std::unique_ptr<int> p3 = std::move(p1);  // p1变为nullptr
std::cout << *p3 << std::endl;//输出结果：42
if (p1 == nullptr)
    std::cout << "Yeah" << std::endl;//输出结果：Yeah

// 自定义删除器
auto deleter = [](int* p) 
{
    std::cout << "Deleting int" << std::endl;
    delete p;
};
std::unique_ptr<int, decltype(deleter)> p4(new int(200), deleter);
```

**b.数组版本**

```cpp
// 管理数组
std::unique_ptr<int[]> arr(new int[5]{1, 2, 3, 4, 5});
arr[0] = 10;  // 支持下标访问

//注：错误实现
//std::unique_ptr<int*> vc = std::make_unique<int*>({ 2,4,6,8,10 });
// std::make_unique<int*>({ 2,4,6,8,10 }); // 错误：std::make_unique 不能这样用

//使用智能指针管理数组只能使用上述形式
```

**c.什么时候用`std::unique_ptr<T>`？**

`std::unique_ptr<T>`比`std::shared_ptr<T>`具有更小的内存，而且不需要维护引用计数，因此它的性能更好。当我们需要一个独占的指针时，应该优先使用`std::unique_ptr<T>`。



### 2.`std::shared_ptr`

**特点**：共享所有权，使用引用计数

`std::shared_ptr<T>`是一个类模板，它的对象行为像指针，但是它还能记录有多少个对象共享它管理的内存对象。

多个`std::shared_ptr<T>`可以共享同一个对象。

当最后一个`std::shared_ptr<T>`被销毁时，它会自动释放它所指向的对象。一个`shared_ptr<T>`指针可以通过`make_shared<T>`函数来创建，也可以通过拷贝或赋值另一个`shared_ptr`来创建。如图所示，sp1和sp2指向同一个对象，内存对象的引用计数为2。当sp1被销毁时，引用计数减为1，sp2仍然指向该对象。当sp2被销毁时，引用计数减为0，内存对象被销毁。

**a.`std::shared_ptr`的底层原理：**

```cpp

element_type*	   _M_ptr;         // Contained pointer.
__shared_count<_Lp>  _M_refcount;    // Reference counter.
```

`std::shared_ptr`在内部维护一个引用计数，其只有两个指针成员，一个指针是所管理的数据的地址；还有一个指针是控制块的地址，包括引用计数、weak_ptr计数、删除器(Deleter)、分配器(Allocator)。因为不同shared_ptr指针需要共享相同的内存对象，因此**引用计数的存储是在堆上的**。而unique_ptr只有一个指针成员，指向所管理的数据的地址。因此一个shared_ptr对象的大小是raw_pointer大小的两倍。

```cpp
// 32位编译器下
std::cout<<sizeof(std::shared_ptr<int>)<<std::endl; // 8
std::cout<<sizeof(std::unique_ptr<int>)<<std::endl; // 4
```

**b.基本使用**

```cpp
#include <memory>
#include <iostream>

class MyClass
{
public:
    MyClass() { std::cout << "MyClass constructed\n"; }
    ~MyClass() { std::cout << "MyClass destroyed\n"; }
    void show() { std::cout << "Hello from MyClass\n"; }
};

// 创建shared_ptr
std::shared_ptr<MyClass> sp1 = std::make_shared<MyClass>();
auto sp2 = sp1;  // 引用计数+1

// 获取引用计数
std::cout << "Use count: " << sp1.use_count() << std::endl;

// 自定义删除器
std::shared_ptr<MyClass> sp3(
    new MyClass,
    [](MyClass* p) {
        std::cout << "Custom delete\n";
        delete p;
    }
);
```

**c.数组版本**

```cpp
// 管理数组（需要自定义删除器）
std::shared_ptr<int> arr( new int[5], std::default_delete<int[]>() );
```

**d.`std::shared_ptr<T>`的内置方法**

| 方法                 | 用途                                                         |
| -------------------- | ------------------------------------------------------------ |
| make_shared<T>(args) | 返回一个shared_ptr，指向一个动态分配的类型为T的对象，使用args初始化此对象。 |
| shared_ptr<T>p(q)    | p是q的拷贝，此操作递增q中的计数器。q中的指针必须能转换为T*。 |
| shared_ptr<T>p = q   | p是q的拷贝，此操作递增q中的计数器。q中的指针必须能转换为T*。 |
| p.unique()           | 如果p.use_count()为1，返回true，否则返回false。              |
| p.use_count()        | 返回与p共享对象的智能指针数量。                              |

```cpp
//程序示例：
std::shared_ptr<int> sp1 = std::make_shared<int>(42);
std::cout<<sp1.unique()<<std::endl; // 1
std::shared_ptr<int> sp2 = sp1;
std::shared_ptr<int> sp3(sp1);
std::shared_ptr<int> sp4(new int(44)); // Not recommended
std::cout<<sp1.use_count()<<std::endl; // 3
sp1.reset();//释放sp1，将sp1置为空
std::cout<<sp1.use_count()<<std::endl; // 0
std::cout<<sp2.use_count()<<std::endl; // 2
```

**e.什么时候用`std::shared_ptr<T>`？**

通常用于一些资源创建昂贵比较耗时的场景， 比如涉及到文件读写、网络连接、数据库连接等。当需要共享资源的所有权时，例如，一个资源需要被多个对象共享，但是不知道哪个对象会最后释放它，这时候就可以使用`std::shared_ptr<T>`。



### 3.`std::shared_ptr` 和 `std::unique_ptr`共有操作

| 方法              | 用途                                     |
| ----------------- | ---------------------------------------- |
| p.get()           | 返回p中保存的指针，不会影响p的引用计数。 |
| p.reset()         | 释放p指向的对象，将p置为空。             |
| p.reset(q)        | 释放p指向的对象，令p指向q。              |
| p.reset(new T)    | 释放p指向的对象，令p指向一个新的对象。   |
| p.swap(q)         | 交换p和q中的指针。                       |
| swap(p, q)        | 交换p和q中的指针。                       |
| p.operator*()     | 解引用p。                                |
| p.operator->()    | 成员访问运算符，等价于(*p).member。      |
| p.operator bool() | 检查p是否为空指针。                      |

```cpp
//程序示例--独占指针：
std::unique_ptr<int> p1 = std::make_unique<int>(42);
std::unique_ptr<int> p2 = std::make_unique<int>(44);
int* p = p1.get();
std::cout<<*p<<std::endl; // 42
p1.swap(p2);
std::cout<<*p1<<std::endl; // 44
std::cout<<*p2<<std::endl; // 42
p1.reset();
std::cout<<p1.get()<<std::endl; // 0, first call get(), then call operator bool()
```



### 4.`std::weak_ptr`

**特点**：弱引用，不增加引用计数，解决循环引用问题

标准库还定义了一个名为`weak_ptr`的伴随类，它是一种弱引用，**指向shared_ptr所管理的对象**，而不影响所指对象的生命周期，也就是将一个`weak_ptr`绑定到一个`shared_ptr`不会改变`shared_ptr`的引用计数。不论是否有`weak_ptr`指向，一旦最后一个指向对象的`shared_ptr`被销毁，对象就会被释放。

**a.如何读取引用对象**

`weak_ptr`对它所指向的`shared_ptr`所管理的对象没有所有权，不能对它解引用，因此若要读取引用对象，必须要转换成`shared_ptr`。 C++中提供了lock函数来实现该功能。如果对象存在，`lock()`函数返回一个指向共享对象的`shared_ptr`，否则返回一个空`shared_ptr`。

**b.如何判断`weak_ptr`指向对象是否存在呢？**

`weak_ptr`提供了一个成员函数`expired()`来判断所指对象是否已经被释放。如果所指对象已经被释放，expired()返回true，否则返回false。

```cpp
//程序示例：
std::shared_ptr<int> sp1(new int(22));
std::shared_ptr<int> sp2 = sp1;
std::weak_ptr<int> wp = sp1; // point to sp1
std::cout<<wp.use_count()<<std::endl; // 2
if(!wp.expired())
{
    std::shared_ptr<int> sp3 = wp.lock();//lock()函数返回一个指向共享对象的shared_ptr
    std::cout<<*sp3<<std::endl; // 22
}
```

**c.`std::weak_ptr` 一些内置方法**

| 方法           | 用途                                                         |
| -------------- | ------------------------------------------------------------ |
| use_count()    | 返回与之共享对象的shared_ptr的数量                           |
| expired()      | 检查所指对象是否已经被释放                                   |
| lock()         | 返回一个指向共享对象的shared_ptr，若对象不存在则返回空shared_ptr |
| owner_before() | 提供所有者基于的弱指针的排序                                 |
| reset()        | 释放所指对象                                                 |
| swap()         | 交换两个weak_ptr对象                                         |

**d.基本使用：**

```cpp
#include <memory>
#include <iostream>

class B;

class A 
{
public:
    std::shared_ptr<B> b_ptr;
    ~A() { std::cout << "A destroyed\n"; }
};

class B 
{
public:
    std::weak_ptr<A> a_ptr;  // 使用weak_ptr避免循环引用
    ~B() { std::cout << "B destroyed\n"; }
};

int main() {
    auto a = std::make_shared<A>();
    auto b = std::make_shared<B>();
    
    a->b_ptr = b;
    b->a_ptr = a;  // 不会增加引用计数
    
    // 使用weak_ptr访问对象
    if (auto shared_a = b->a_ptr.lock()) {
        shared_a->b_ptr.reset();
    }
}
```

- 什么是循环引用问题？

  循环引用是指两个或多个对象之间通过`shared_ptr`相互引用，形成了一个环，导致它们的引用计数都不为0，从而导致内存泄漏。

## 三.最佳实践和使用注意事项

### 1.优先使用`make_shared`和`make_unique`

```cpp
// 推荐
auto p1 = std::make_shared<MyClass>();
auto p2 = std::make_unique<MyClass>();

// 不推荐（可能造成内存泄漏）
std::shared_ptr<MyClass> p3(new MyClass());
```

### 2. 避免循环引用

```cpp
// 错误示例：循环引用导致内存泄漏
class Node
{
public:
    std::shared_ptr<Node> next;  // 应使用weak_ptr
};

// 正确做法
class SafeNode 
{
public:
    std::weak_ptr<SafeNode> next;
};
```

### 3. 不要混用原始指针和智能指针

```cpp
// 危险做法
int* raw_ptr = new int(42);
std::shared_ptr<int> sp1(raw_ptr);
std::shared_ptr<int> sp2(raw_ptr);  // 双重释放！

// 正确做法
auto sp1 = std::make_shared<int>(42);
auto sp2 = sp1;  // 共享所有权
```

### 4. 谨慎使用`get()`获取原始指针

```cpp
std::shared_ptr<int> sp = std::make_shared<int>(42);
int* raw = sp.get();

// 危险：不要用这个原始指针创建新的智能指针
// std::shared_ptr<int> sp2(raw);  // 错误！
// 危险：不要delete这个指针
// delete raw;  // 错误！
```

### 5. 注意多线程安全性

- `shared_ptr`的引用计数操作是原子的
- 但指向的对象本身不是线程安全的

```cpp
#include <memory>
#include <thread>

std::shared_ptr<int> global_sp = std::make_shared<int>(0);

void increment() {
    for (int i = 0; i < 1000; ++i) {
        // 需要额外的同步机制保护*global_sp
        auto local_copy = global_sp;  // 安全：引用计数操作
        // 操作local_copy指向的内容需要同步
    }
}
```

### 6. 自定义删除器的使用场景

```cpp
// 管理文件指针
#include <cstdio>
auto file_deleter = [](FILE* fp) {
    if (fp) fclose(fp);
};
std::unique_ptr<FILE, decltype(file_deleter)> file_ptr(fopen("test.txt", "r"), file_deleter);

// 管理数组
std::unique_ptr<int[], void(*)(int*)> array_ptr(
    new int[10],
    [](int* p) { delete[] p; }
);
```

### 7. 性能考虑

- `unique_ptr`几乎没有开销
- `shared_ptr`有引用计数的开销
- `make_shared`比直接构造`shared_ptr`更高效

### 8. 类型转换

```cpp
class Base {
public:
    virtual ~Base() = default;
};

class Derived : public Base {};

// 向下转型
std::shared_ptr<Base> base_ptr = std::make_shared<Derived>();
auto derived_ptr = std::dynamic_pointer_cast<Derived>(base_ptr);

if (derived_ptr) {
    // 转换成功
}
```

### 9. 常见错误模式

```cpp
// 错误1：同一原始指针创建多个智能指针
int* p = new int(10);
std::shared_ptr<int> sp1(p);
std::shared_ptr<int> sp2(p);  // 运行时错误

// 错误2：返回局部智能指针的引用
std::shared_ptr<int>& bad_function() {
    auto sp = std::make_shared<int>(42);
    return sp;  // 返回悬空引用
}

// 错误3：循环引用
class BadClass {
    std::shared_ptr<BadClass> other;  // 应使用weak_ptr
};
```

## 四、总结

### 选择指南：

1. **`unique_ptr`**：独占所有权，不需要共享时使用
2. **`shared_ptr`**：需要共享所有权时使用
3. **`weak_ptr`**：打破循环引用或观察对象时使用

### 黄金法则：

1. 优先使用`make_unique`和`make_shared`
2. 明确所有权关系
3. 避免原始指针和智能指针混用
4. 使用`weak_ptr`解决循环引用
5. 注意智能指针的生命周期
